## Acceptance Criteria
- When person A assigns a chore to person B, a push notification is sent to all of person B's subscribed devices
- When person B completes a chore assigned by person A, person A receives a completion push notification
- When person A assigns a reminder to person B, person B receives a push notification with the reminder details
- When a reminder becomes due, the assigned person receives a due-alert push notification
- When person A adds a grocery item and tags person B, person B receives a notification
- Push notifications are NOT sent to the person who performed the action (no self-notification)
- If person B has no push subscription, the system handles gracefully without errors
- Notification bell endpoint returns unread notification count correctly for a given user
- Notification bell count decrements or clears after notifications are marked as read
- Multiple household members each receive their respective notifications independently
- Push subscribe endpoint stores subscription and returns success for a valid user
- Push send endpoint rejects requests with missing or invalid payload
- All notification API routes return appropriate HTTP status codes (200, 400, 401, 500)
