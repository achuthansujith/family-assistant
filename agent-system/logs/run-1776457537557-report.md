## Acceptance Criteria
- When Person A adds a chore assigned to Person B, a push notification is sent to all of Person B's subscribed devices with the correct title and body mentioning the chore name and assigner
- When Person B marks a chore as complete, Person A receives a push notification confirming completion with the chore name and completer's name
- When Person A adds a reminder assigned to Person B, Person B receives a push notification for the new reminder assignment
- When a reminder becomes due, the assigned person receives a due-alert push notification
- When Person A adds a grocery item and assigns it to Person B, Person B is notified of the new grocery assignment
- When Person A creates a household event, all household members receive an event notification
- When a meal plan is updated and assigned to Person B, Person B receives a meal assignment notification
- Notifications are only sent to members within the same household and never to users outside the household
- The notification bell API returns unread notification count and list correctly for the authenticated user
- The notification bell component renders the correct unread badge count and clears on open
- The push subscribe API correctly saves and updates push subscriptions per user in Supabase
- The push send API returns 401 for unauthenticated requests and 400 for malformed payloads
- usePushNotifications hook requests permission, subscribes to push, and calls the subscribe API on mount when permission is granted
- usePushNotifications hook does not throw and gracefully handles browsers without push support
- Notification payloads include a deep-link URL so tapping the notification navigates to the relevant page
- All tests pass in CI without requiring live Supabase or web-push credentials by using mocks and MSW handlers
